Разработка клиент-серверного приложения на JavaScript

## Введение

Клиент-серверное приложение состоит из двух частей:

*   **Frontend (интерфейс)**: создается с помощью HTML, CSS, JS и React.
*   **Backend (бизнес-логика)**: состоит из базы данных, сервера (машины) и серверного приложения.

## PERN - технологический стек разработки

*   **PostgresSQL**: система управления базами данных.
*   **Express**: фреймворк для создания веб-серверов.
*   **React**: библиотека для создания пользовательских интерфейсов.
*   **Node.JS**: платформа, позволяющая языку JS работать вне браузера.

## Node.JS

*   **Асинхронность и событийно-ориентированная архитектура**: основные параметры Node.JS.
*   **Не блокирующий ввод-вывод**: сервер не ждет завершения операции, вместо этого он продолжает выполнять другой код.
*   **Однопоточность с циклом событий (event loop)**: механизм, который постоянно проверяет, завершились ли какие-нибудь асинхронные операции.

## Асинхронные операции

Рассмотрим пример ресторана:

*   повара готовят блюда параллельно;
*   каждый повар смотрит на свой заказ и начинает готовить;
*   как только блюдо готово, его сразу относят клиенту.

## ORM - sequelize

*   **Object-Relational Mapping**: работа с моделями как с объектами.

## Основные понятия

*   **Бизнес-логика**: сценарии обработки клиентских запросов.
*   **Стрелочные функции + массивы + DOM**: используемые технологии.


## NPM - менеджер пакетов

*   **Node Packages Module**: менеджер пакетов, который позволяет легко установить и использовать в своем проекте открытые библиотеки и инструменты.

## Этапы разработки серверной части на Node.JS

### 1. Подготовка и инициализация проекта

*   Создание директорий проекта (папки: клиент, сервер).
*   Инициализация проекта (npm init).
*   Установка необходимых зависимостей (модулей).

### 2. Создание базового HTTP-сервера с помощью Express

*   Импорт фреймворка Express, создание экземпляра приложения и определение порта, на котором сервер будет принимать входящие подключения.
*   Настройка переменных окружения.
*   Создание файла конфигурации БД (исполняемый файл, который подключает сервер к БД).
*   Настройка промежуточного ПО (Midleware), описание моделей данных (с помощью Sequelize).
